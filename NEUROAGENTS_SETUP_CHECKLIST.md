# Чек-лист настройки нейро-сметчика в NeuroAgents

## 1. Назначение

Этот чек-лист нужен, чтобы по шагам перенести подготовленную систему в `app.neuroagents.pro` и не потерять важные элементы:

- роль агента
- промпт
- база знаний
- переменные
- логика передачи менеджеру
- тесты
- интеграция с таблицей/CRM

Главная цель:

> Запустить не просто чат-бота, а первичного ИИ-сметчика, который собирает заявку, не выдумывает цену и передает данные в понятном формате.

## 2. Подготовленные файлы

Перед настройкой должны быть готовы:

```text
NEUROAGENTS_INTEGRATION_PLAN.md
NEUROAGENTS_AGENT_PROMPT.md
NEUROAGENTS_KNOWLEDGE_BASE.md
NEUROAGENTS_TEST_CASES.md
NEUROAGENTS_VARIABLES_AND_FIELDS.md
```

Назначение:

- `NEUROAGENTS_INTEGRATION_PLAN.md` — общая архитектура.
- `NEUROAGENTS_AGENT_PROMPT.md` — промпт и сценарии поведения.
- `NEUROAGENTS_KNOWLEDGE_BASE.md` — база знаний.
- `NEUROAGENTS_TEST_CASES.md` — приемочные тесты.
- `NEUROAGENTS_VARIABLES_AND_FIELDS.md` — карта переменных и полей.

## 3. Рекомендуемый порядок настройки

```text
1. Создать агента.
2. Задать роль и системную инструкцию.
3. Загрузить/добавить базу знаний.
4. Создать переменные.
5. Настроить сценарий сбора заявки.
6. Настроить передачу менеджеру.
7. Настроить интеграцию с таблицей/CRM.
8. Прогнать тест-кейсы.
9. Исправить ошибки.
10. Запускать MVP.
```

## 4. Шаг 1. Создать агента

Название агента:

```text
Нейро-сметчик: террасы и веранды
```

Внутреннее описание:

```text
Первичный ИИ-сметчик для заявок на строительство и монтаж экстерьера: террасы, веранды, навесы, крыльцо, пристройки. Собирает вводные, определяет режим расчета, формирует карточку заявки и передает менеджеру.
```

Публичное описание, если нужно:

```text
Помогу собрать данные для предварительной сметы и передать заявку специалисту для проверки.
```

## 5. Шаг 2. Вставить системный промпт

Основной источник:

```text
NEUROAGENTS_AGENT_PROMPT.md
```

Что вставлять:

1. Если интерфейс позволяет длинный промпт — использовать раздел `2. Системный промпт`.
2. Если есть ограничение по объему — использовать раздел `19. Короткая инструкция для вставки в NeuroAgents`.

Критически важно, чтобы в промпте были правила:

- не выдумывать цены
- не давать точную стоимость без проверки
- не выбирать конструкцию за специалиста
- задавать вопросы короткими шагами
- формировать карточку заявки
- передавать сложные случаи менеджеру

Проверка после вставки:

```text
Задать агенту тестовый вопрос: "Сколько стоит веранда?"
```

Ожидаемо:

- агент не называет цену
- объясняет, что нужны размеры и состав работ
- задает 3-5 уточняющих вопросов

## 6. Шаг 3. Добавить базу знаний

Основной источник:

```text
NEUROAGENTS_KNOWLEDGE_BASE.md
```

Если в NeuroAgents можно загружать базу блоками, добавлять отдельными разделами:

```text
1. Главный принцип сметчика
2. Типы объектов
3. Состав работ
4. Материалы
5. Коэффициенты и условия объекта
6. Работа с проектом
7. Работа с чужой сметой
8. Частые вопросы клиентов
9. Возражения и ответы
10. Красные флаги
```

Если база знаний загружается одним документом:

- загрузить весь файл
- убедиться, что агент использует его в ответах
- проверить на 2-3 вопросах

Проверка:

```text
Клиент: "У меня участок на уклоне и надо снести старое крыльцо"
```

Ожидаемо:

- агент видит риски
- не считает автоматически
- просит фото/размеры
- предлагает передать специалисту

## 7. Шаг 4. Создать переменные

Основной источник:

```text
NEUROAGENTS_VARIABLES_AND_FIELDS.md
```

### Минимальные переменные MVP

Создать в первую очередь:

```text
client_name
phone
city_or_location
object_type
work_type
calculation_mode
length_m
width_m
area_m2
foundation_type
deck_material
roof_required
has_project_file
missing_data
risk_flags
manager_comment
next_step
```

### Дополнительные переменные после MVP

Добавить позже:

```text
glazing_required
roof_area_m2
glazing_area_m2
access_complexity
urgency
budget_range
estimate_confidence
project_file_type
dimensions_confidence
decision_stage
```

### Проверка переменных

Прогнать диалог:

```text
Клиент: "Андрей. Терраса 6 на 4, ДПК, Екатеринбург. Фундамент не знаю."
```

Ожидаемые значения:

```text
client_name = Андрей
object_type = terrace
length_m = 6
width_m = 4
area_m2 = 24
deck_material = wpc
city_or_location = Екатеринбург
foundation_type = unknown
missing_data включает foundation_type
risk_flags включает unknown_foundation
```

## 8. Шаг 5. Настроить сценарий сбора заявки

Сценарий должен идти не анкетой на 20 вопросов, а короткими блоками.

### Блок 1. Определение объекта

Вопрос:

```text
Что нужно рассчитать: террасу, веранду, навес, крыльцо, пристройку или другой объект?
```

Цель:

```text
object_type
object_type_raw
work_type
```

### Блок 2. Размеры

Вопрос:

```text
Какие примерные размеры? Достаточно длины и ширины, например 6 × 4 м.
```

Цель:

```text
length_m
width_m
area_m2
dimensions_confidence
```

### Блок 3. Состав работ

Вопрос:

```text
Что должно входить в расчет: только настил/каркас или еще фундамент, кровля, лестница, ограждение, остекление?
```

Цель:

```text
foundation_required
roof_required
glazing_required
railing_required
stairs_required
demolition_required
```

### Блок 4. Материалы

Вопрос:

```text
По материалам уже есть пожелания: лиственница, ДПК, сосна, металл, металлочерепица, поликарбонат?
```

Цель:

```text
deck_material
frame_material
roof_material
material_comment
```

### Блок 5. Фундамент

Вопрос:

```text
Какой фундамент планируется или уже есть: сваи, плита, существующее основание, другое?
```

Если клиент не знает:

```text
foundation_type = unknown
risk_flags включает unknown_foundation
missing_data включает foundation_type
```

### Блок 6. Локация и доступ

Вопрос:

```text
Где находится объект и есть ли особенности доступа: далеко от города, узкий проход, нет подъезда, уклон участка?
```

Цель:

```text
city_or_location
access_complexity
access_comment
delivery_distance_km
```

### Блок 7. Проект/фото

Вопрос:

```text
Есть ли проект, чертеж, фото участка или пример того, что хотите получить?
```

Цель:

```text
has_project_file
project_file_type
file_received
```

### Блок 8. Контакт

Запрашивать, когда есть смысл передать заявку:

```text
Чтобы специалист мог уточнить детали и вернуться с расчетом, оставьте, пожалуйста, номер телефона для связи.
```

Цель:

```text
phone
client_name
preferred_contact_time
```

## 9. Шаг 6. Настроить стоп-правила

Агент должен останавливать точный расчет, если:

```text
Нет размеров.
Нет состава работ.
Неизвестен фундамент.
Нет локации.
Клиент просит точную цену сразу.
Есть сложный участок.
Есть демонтаж.
Есть теплая пристройка.
Есть проект с несколькими вариантами.
Есть остекление без размеров.
```

Обязательная фраза:

```text
Чтобы не назвать случайную сумму, точную цену сейчас лучше не фиксировать. Я соберу предварительную структуру расчета и отмечу, что нужно проверить специалисту.
```

## 10. Шаг 7. Настроить передачу менеджеру

Передавать менеджеру, если:

```text
phone заполнен
или клиент просит звонок
или есть проект
или есть сложная заявка
или клиент готов заказать
или есть точный запрос на расчет
```

Формат сообщения менеджеру:

```text
Новая заявка на расчет.

Клиент: {{client_name}}
Телефон: {{phone}}
Локация: {{city_or_location}}

Объект: {{object_type_raw}}
Тип работ: {{work_type}}
Режим расчета: {{calculation_mode}}

Размеры:
- длина: {{length_m}} м
- ширина: {{width_m}} м
- площадь: {{area_m2}} м2

Состав:
- фундамент: {{foundation_type}}
- настил: {{deck_material}}
- кровля: {{roof_required}}
- остекление: {{glazing_required}}
- ограждение: {{railing_required}}
- лестница: {{stairs_required}}
- демонтаж: {{demolition_required}}

Условия:
- доступ: {{access_complexity}}
- комментарий: {{access_comment}}

Не хватает:
{{missing_data}}

Риски:
{{risk_flags}}

Комментарий агента:
{{manager_comment}}

Следующий шаг:
{{next_step}}
```

## 11. Шаг 8. Настроить Google Sheets / n8n

### Для ручного MVP

Сначала можно без автоматического подключения:

```text
NeuroAgents -> карточка заявки -> менеджер вручную переносит в таблицу
```

Это нормально для первых тестов.

### Для автоматизации через n8n

Передавать JSON из:

```text
NEUROAGENTS_VARIABLES_AND_FIELDS.md
```

Минимальная строка для Google Sheets:

```text
created_at
client_name
phone
city_or_location
object_type
work_type
calculation_mode
length_m
width_m
area_m2
foundation_type
deck_material
roof_required
glazing_required
has_project_file
access_complexity
urgency
missing_data
risk_flags
estimate_confidence
next_step
manager_comment
lead_status
```

### Когда подключать автоматизацию

Не сразу.

Сначала нужно:

```text
1. Прогнать тесты.
2. Получить 10-20 реальных заявок.
3. Посмотреть, какие поля реально нужны.
4. Только потом фиксировать интеграцию.
```

## 12. Шаг 9. Прогнать тест-кейсы

Основной источник:

```text
NEUROAGENTS_TEST_CASES.md
```

Минимум перед запуском:

```text
Тест 1. Простая терраса
Тест 2. Цена без данных
Тест 4. Неизвестный фундамент
Тест 5. Готовый проект PDF
Тест 8. Сложный участок
Тест 9. Требует точную цену
Тест 13. Теплая пристройка
```

Критерий:

```text
Ни в одном из этих тестов агент не должен назвать точную цену без проверки.
```

## 13. Шаг 10. Исправить ошибки после тестов

Если агент ошибся, определить тип ошибки:

```text
1. Ложная точность.
2. Слишком много вопросов.
3. Слишком мало вопросов.
4. Не понял тип объекта.
5. Не передал менеджеру.
6. Не отметил риск.
7. Смешал варианты.
8. Дал слишком технический ответ.
```

Что делать:

```text
1. Добавить правило в базу знаний.
2. Уточнить системный промпт.
3. Добавить переменную, если данные теряются.
4. Добавить тест-кейс.
5. Повторить проверку.
```

## 14. Шаг 11. MVP-запуск

Запускать можно, если:

```text
1. Агент прошел минимум 12 из 15 тестов.
2. Нет критических ошибок в рискованных сценариях.
3. Агент не выдумывает цену.
4. Агент формирует понятную карточку.
5. Менеджер понимает, что делать с заявкой.
6. Есть таблица или ручной процесс расчета.
```

Первый канал для запуска:

```text
Telegram или виджет на сайте
```

Почему:

- проще тестировать
- легче смотреть переписки
- быстрее править сценарий
- меньше риска, чем сразу подключать все каналы

## 15. Минимальный процесс после запуска

Каждую неделю фиксировать:

```text
1. Количество диалогов.
2. Количество собранных заявок.
3. Количество заявок, переданных менеджеру.
4. Сколько заявок были неполными.
5. Где агент ошибся.
6. Какие вопросы клиенты задают чаще всего.
7. Какие поля не собираются.
8. Какие ответы нужно улучшить.
```

## 16. Метрики MVP

Минимальные метрики:

```text
conversion_to_contact
qualified_leads_count
manager_handoff_count
incomplete_leads_count
critical_error_count
average_questions_before_contact
manual_processing_time_saved
```

Рабочие целевые ориентиры:

```text
critical_error_count = 0
conversion_to_contact растет
manual_processing_time_saved >= 50%
80% карточек понятны менеджеру без чтения всей переписки
```

## 17. Что не делать на старте

Не делать сразу:

- сложный API
- полную автоматическую смету без менеджера
- расчет всех типов объектов
- интеграцию со всеми каналами
- слишком длинный сценарий
- финальную цену без проверки
- автоматический разбор сложных PDF как основной сценарий

Лучший старт:

```text
террасы и веранды -> ручной сбор вводных -> карточка заявки -> менеджер/таблица -> тесты -> улучшение
```

## 18. Порядок развития после MVP

### Этап 1

Ручной MVP:

```text
диалог -> карточка -> менеджер -> таблица
```

### Этап 2

Полуавтоматизация:

```text
диалог -> переменные -> Google Sheets -> менеджер
```

### Этап 3

Расчетный модуль:

```text
диалог -> n8n/API -> расчет -> предварительный ответ -> менеджер
```

### Этап 4

Расширение:

```text
добавить навесы, крыльцо, проверку чужих смет, проектные PDF
```

## 19. Финальная проверка перед реальным трафиком

Перед запуском убедиться:

```text
[ ] Агент создан
[ ] Промпт вставлен
[ ] База знаний добавлена
[ ] Переменные созданы
[ ] Сценарий вопросов работает
[ ] Стоп-правила работают
[ ] Карточка заявки формируется
[ ] Менеджер получает понятный вывод
[ ] Тесты пройдены
[ ] Есть процесс расчета
[ ] Есть человек, который проверяет спорные заявки
```

## 20. Следующий артефакт

После чек-листа настройки нужен расчетный слой:

```text
NEURO_SMETCHIK_SPREADSHEET_SPEC.md
```

Он должен описывать:

- структуру Google Sheets / Excel
- листы
- поля
- формулы
- прайс
- коэффициенты
- расчетную логику
- клиентскую версию сметы
- проверку менеджера

Без таблицы агент останется хорошим сборщиком заявок, но не станет сметной системой.
